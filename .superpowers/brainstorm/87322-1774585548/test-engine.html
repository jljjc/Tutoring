Question Assembly Flow

1. Student starts a test
↓
2. System checks question bank — are there enough unused questions for this student?
↓
YES
Pull from bank,
randomise order
NO
Generate fresh via
Claude API, cache result
↓
3. Questions shuffled + delivered one at a time with timer
↓
4. Each answer saved in real-time to test_answers
↓
5. Test complete → score calculated → report generated → tutoring triggered for wrong answers

Anti-repeat guarantee: The system tracks which questions a student has already seen. Claude is only called when the bank runs low — keeping API costs down while ensuring fresh questions for every session.
